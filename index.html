04. Landmarks and sectioning elements used

Use ARIA landmark roles or HTML5 sectioning elements to describe document structure and outline different parts of the page, such as navigation, main content and footer info.

Landmark roles include: banner, navigation, search, main, complementary, article, section, and contentinfo.

Equivalent HTML5 elements include: <header>, <nav>, <main>, <aside>, <article>,<section>, and <footer>. These elements have default landmark roles and do not require an ARIA role attribute to be set. To support older browsers though, it is still recommended to provide both options as support for ARIA and HTML5 is not always paralleled.

For more information refer to the W3C HTML5 Sectioning Elements: ARIA Landmarks Example.

07. Search with NAV element, specific role attribute and a proper label

The search (together with the main menu) is often the preferred way of navigating a site. To improve quick access always wrap the main search in a <nav> element (landmark) with role="search". To provide a descriptive name for the interactive elements always add a <label> element with a for="" attribute to the input and a name for the button. Hiding may be done with a screen reader only class.

08. Headings marked up as such

The template suggests styling for headings using the corresponding markup within the HTML code.

Other things to consider, in order to make headings more usable for screen reader users:

  • Lists and headings used together to mark up content should be avoided
  • For a correct heading outline it is sometimes necessary to hide them visually
  • Headings should be coded around content that doesn’t change, i.e. a category name such as ‘Banking News’ rather than a news item that will update frequently

09. Image with text alternative

This image of the "Infographic" is considered informative, so a text alternative has been provided to briefly describe the image.

The element type or trait, such as image or button, does not need to be included in the alternative as it is programmatically determined and added by the screen reader. For example, an image coded as an image with an alternative beginning "Image of ..." would be spoken as "Image. Image of ...".

13. Links are Links and Buttons are Buttons

How to choose between a link and a button:

  1. Will this control be used to initiate an immediate action? It’s a button!
  2. Is the action to navigate to another page? It’s a link!

General guidelines on making the choice:

  • Use command buttons for:
    • Primary commands
    • Displaying windows used to gather input or making choices, even if they are secondary commands
    • Destructive or irreversible actions, as well as commitment within wizards and page flows
  • Use links for navigation to another page, window, or Help topic; display definitions; and command menus

15. Sufficient color contrast

Contrast ratio between foreground (text) color (#333333) and background color (#ffffff) is with 12.6:1 sufficiently high for most people to be able to differentiate.

  • For text or images of text avoid background colours or use background colours that have sufficient contrast from the foreground colour
  • If background colours are used, apply the 4.5:1 contrast ratio to foreground colours for regular text and 3:1 contrast ratio for big or bold text.

Note! Improvements on the template are propagated throughout the entire website

HTML Example


.card { background-color: #ffffff; }
.lead { color: #333333; }

WCAG Success Criteria

  • 1.4.3 Contrast (Minimum) Level AA

    Situation A: text is less than 18 point if not bold and less than 14 point if bold

    • G18: Ensuring that a contrast ratio of at least 4.5:1 exists between text (and images of text) and background behind the text
    • G148: Not specifying background color, not specifying text color, and not using technology features that change those defaults

    Situation B: text is at least 18 point if not bold and at least 14 point if bold

    • G145: Ensuring that a contrast ratio of at least 3:1 exists between text (and images of text) and background behind the text
    • G148: Not specifying background color, not specifying text color, and not using technology features that change those defaults
